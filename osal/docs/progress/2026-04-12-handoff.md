# OSAL Handoff

日期：2026-04-12

## 当前目标

本轮工作针对 `Middleware/osal` 做边界收口，不是继续加功能：

- 统一等待状态码语义
- 区分 `OSAL_ERR_RESOURCE`、`OSAL_ERR_BLOCKED`、`OSAL_ERR_DELETED`
- 清理 system/platform 边界里的 `osal_platform_init()` weak 残留
- 同步示例、README、CHANGELOG 和相关文档
- 把工程重新收敛到 `0 error / 0 warning`

## 不是半成品，但属于明确边界

这些是当前设计选择，不是这轮遗漏：

- 调度器是协作式，不是抢占式
- `mutex` 仍是最小实现，没有 owner 检查，也没有优先级继承
- `event` 的等待只支持任务态，ISR 只允许 `set/clear`
- `queue` 是固定项大小，不支持变长消息
- `platform/example/stm32f4` 里的 LED weak 钩子仍保留在示例层，不属于 system 残留
